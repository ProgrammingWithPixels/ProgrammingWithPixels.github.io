Goal of PwP

    The ultimate goal of PwP is to enable generalist software engineering agents, that can solve a range of software engineering tasks, without hand-engineered priors in the form of action, observation and tool space. Agents should be capable of operating in environments similar to that of a human software engineer.

The Limits of Tool-Based Agents

Current AI agents for software engineering rely on hand-engineered tool APIs, limiting their generalization across tasks and languages. These approaches require extensive human engineering for each new capability and struggle with multimodal programming tasks.

The PwP Paradigm

PwP enables agents to interact with IDEs through visual perception and basic actions:

  • Visual Interaction: Observe IDE state through screenshots
  • Primitive Actions: Typing, clicking, navigation
  • Tool Agnostic: Leverage existing IDE capabilities naturally

Key Advantages

  • IDE-Centric Design: Full access to VSCode tools through visual interaction
  • PwP-Bench: Unified benchmark spanning 15 tasks across 8 languages
  • Generalist Agents: Single agent architecture across all tasks
  • Critical Insights: Current models struggle with visual grounding and tool discovery
